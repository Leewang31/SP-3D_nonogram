타겟 플랫폼

플랫폼이 기술 스택과 조작 방식 전부를 결정함

A

모바일 (iOS / Android)

터치 기반 조작 — 탭으로 블록 제거, 핀치로 카메라 줌, 스와이프로 회전

추천 스택: Flutter + Flame 또는 Unity (모바일 빌드)

장점

  • 닌텐도 원작과 동일한 터치 경험
  • 배포 가능 (앱스토어)

고려사항

  • 3D 렌더링 모바일 최적화 필요
  • 빌드 환경 세팅 비용
B

웹 브라우저

마우스/터치 겸용 — 클릭으로 제거, 드래그로 카메라 회전

추천 스택: Three.js + TypeScript 또는 Babylon.js

장점

  • 빌드 없이 즉시 실행·공유
  • 프로토타입 이터레이션 가장 빠름
  • 별도 설치 불필요

고려사항

  • 모바일 터치 UX 추가 작업 필요
C

Godot 4 (데스크탑 + 모바일 export)

GDScript로 구현 — 데스크탑 먼저, 이후 Android/iOS export

스택: Godot 4 + GDScript (animal-hide-game과 동일)

장점

  • 기존 프로젝트(동물숨기)와 스택 통일
  • 3D 렌더링 내장, 멀티플랫폼 export

고려사항

  • Godot 모바일 export 세팅 복잡
