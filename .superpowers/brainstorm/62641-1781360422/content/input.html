입력 처리

터치 이벤트 분류 → 카메라 / 블록 선택 / 레이어 이동

CameraController.gd — 터치 상태 머신
터치 1개
BEGAN → 시작점 기록
MOVED → delta 계산 → 카메라 오빗
ENDED → 이동량 < TAP_THRESHOLD?
→ YES: 레이캐스트 발사 → 블록 hit
→ NO: 오빗 종료
터치 2개 (핀치)
BEGAN → 두 터치 거리 기록
MOVED → 거리 변화 → 카메라 줌
ENDED → 줌 종료
레이캐스트 → 블록 제거 흐름
탭 위치 → Camera.project_ray_origin/direction → PhysicsRayQueryParameters3D → space_state.intersect_ray()
hit 없음 → 무시
hit 있음 → collider.get_meta("grid_pos") → (x, y, z)
→ puzzle_model.remove_block(x, y, z)
OK → block_grid.hide_block(x,y,z) + 이펙트
WRONG → 페널티 +1 + 진동 피드백
ALREADY → 무시
레이어 슬라이더 (CanvasLayer)
HSlider.value_changed → active_layer: int 업데이트
BlockGrid: active_layer 이하 블록만 visible = true
# active_layer = size-1 이면 전체 표시 (기본값)
⚠ 슬라이더 이동 중 레이캐스트 비활성화
# UI 터치와 3D 터치 충돌 방지
TAP_THRESHOLD = 10px. 이 이하 이동은 탭으로 판정. 오빗 중 실수 제거 방지.
