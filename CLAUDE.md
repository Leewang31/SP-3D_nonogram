# CLAUDE.md

이 파일은 Claude Code(claude.ai/code)가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

3D 노노그램 — 프로토타입/데모. 기존 2D 피크로스(노노그램)를 3D 공간으로 확장: NxNxN 격자에서 세 축(X·Y·Z)의 숫자 단서를 보고 어느 셀이 채워졌는지 추론하는 퍼즐 게임.

**현황:** Godot 4 + GDScript. 모바일(iOS/Android). 설계 완료, 구현 진행 중.

## 세션 시작 규칙

**모든 세션은 wiki 확인으로 시작한다.**
1. `index.md` 읽기
2. `wiki/projects/3d-nonogram.md` 읽기
3. `log.md` 마지막 5개 항목 확인
4. 작업 컨텍스트 파악 후 실행

## 3D 노노그램이란

- 기존 노노그램: 2D 격자, 행·열마다 연속 채움 구간 단서
- 3D 확장: NxNxN 복셀 격자, X축 행·Y축 열·Z축 기둥 방향으로 각각 단서 제공
- 각 셀은 채워짐 / 비어있음 / 미확인 세 상태
- 세 축 교차 논리 추론으로 풀이

## 핵심 아키텍처 (기술 스택 무관)

세 책임 분리 유지:

1. **퍼즐 모델** — 순수 데이터: 격자 상태(`filled | empty | unknown`), 축별 단서 집합, 유효성 검증. 렌더링 없음. 독립적으로 테스트 가능해야 함.
2. **렌더러** — 3D 격자 시각화. 핵심 결정: 카메라 조작(오빗), 셀 선택 피드백, 각 면에 단서 패널 표시 방식.
3. **인터랙션** — 사용자 입력 → 셀 상태 변경 매핑, 실행 취소/다시 실행 스택.

## 기술 스택

- **엔진:** Godot 4 + GDScript
- **플랫폼:** 모바일 (iOS / Android)
- **퍼즐 포맷:** JSON (`puzzles/*.json`)
- **씬:** `Main.tscn` 단일 씬 (프로토타입)
- **테스트:** GDScript 독립 실행 (`PuzzleModel.gd` 노드 없음)

## 퍼즐 데이터 포맷 (제안)

```json
{
  "size": 3,
  "solution": [[[1,0,1],[0,1,0],[1,0,1]], ...],
  "clues": {
    "x": [[...], ...],
    "y": [[...], ...],
    "z": [[...], ...]
  }
}
```

단서 생성은 솔루션에서 결정론적·순수 함수로 도출. 사이드이펙트 없어야 함.

## LLM Wiki 운영 규칙

이 프로젝트는 LLM wiki 패턴으로 지식을 관리한다.

### 폴더 구조
| 폴더 | 용도 |
|---|---|
| `wiki/projects/` | 프로젝트 위키 (LLM 생성·유지) |
| `wiki/concepts/` | 개념 페이지, 합성 노트 |
| `wiki/knowledge/` | 레퍼런스, 영구 지식 |
| `raw/` | 원본 소스 — 읽기 전용. 절대 수정 금지 |
| `index.md` | 볼트 전체 카탈로그 (항상 최신 유지) |
| `log.md` | 추가 전용 운영 로그 |

### 모든 wiki 노트 필수 규칙
1. 프론트매터: `date`, `type`, `tags`, `ai-first: true`
2. 첫 섹션: `## 미래의 Claude를 위해` — 2-3문장 요약
3. 외부 사실에 날짜 마커: `(as of YYYY-MM)`
4. 사람/프로젝트/개념 언급 시 `[[wikilink]]` 사용

### 대화 후 저장 규칙
